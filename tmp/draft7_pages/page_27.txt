                                     CHAPTER 6
                                 METHODOLOGY


The development of ImpactLedger follows an Agile methodology, organized into
iterative sprints. Each sprint involves planning, development, testing, and review
phases, allowing the team to incorporate feedback progressively and adapt to
changing requirements. This approach is well-suited for a community-engaged
project where user needs evolve through ongoing interaction with the partner
organization, Lions Club Nagpur Sawaj. Regular review sessions with NGO
coordinators helped refine features and prioritize development tasks.


The technology stack was selected based on criteria of community adoption,
documentation availability, and suitability for the project's requirements. The
frontend is developed using React with Next.js to enable server-side rendering,
improving page load performance and search engine visibility. The backend is
implemented in Node.js with the Express framework, providing a lightweight and
efficient server environment for handling API requests. MongoDB is used as the
primary database for its document-oriented structure, which aligns well with the
hierarchical nature of project and activity data. Firebase is considered as an
alternative for its real-time synchronization capabilities.


Data collection for understanding user needs was carried out through structured
surveys distributed to NGO volunteers and administrators, as well as informal
interviews with Lions Club Nagpur Sawaj members. The surveys covered topics
such as current record-keeping practices, pain points in project management, and
expectations from a digital system. The collected responses informed the
prioritization of features and the design of the user interface, ensuring that the
platform reflects actual operational requirements rather than assumptions.
